# 2026-04-04 Cloud Runタイムアウト設定不足による処理失敗

## 概要

**SEV**: SEV-3

Cloud Runのデフォルトタイムアウト（600s）が動画生成処理の実際の所要時間に対して不足しており、処理が途中で強制終了されていた。フロントエンドのポーリングタイムアウトも短く、ユーザーが途中で「失敗」と誤認する状態だった。

## タイムライン

- 動画生成の統合テスト中、処理が完了する前にCloud Runが接続を切断する事象を確認
- Cloud Runのタイムアウトをデフォルト600sから3600sに引き上げ、フロントエンドのポーリングも60分に延長
- その後、GCSダウンロード修正の過程でタイムアウト値を精査
- 正常系の所要時間を計測した結果、GCSダウンロードタイムアウトは180sが過剰と判明
- 30sに短縮し、フロントエンドポーリングも20分に短縮して適正化

## 根本原因

初期設定がCloud Runのデフォルト値のままで、動画生成・FFmpeg処理・Veo API待機を含む実際の処理時間を考慮していなかった。タイムアウト値に根拠がなく、実測値に基づいていなかった。

## 影響範囲

- **重大度**: SEV-3
- **影響ユーザー数**: 0人（開発中・未リリース機能）
- **ダウンタイム**: 処理が断続的に失敗（数日間）
- **TTD（検知まで）**: テスト中に即時確認
- **TTR（復旧まで）**: 即日（設定変更のみ）
- **データ損失**: なし
- **外部影響**: なし

## 解決策

Cloud Runタイムアウトを3600sに引き上げ。その後、各処理ステップの実測値をもとにタイムアウト値を適正化。

## 再発防止策

- [ ] タイムアウト値は実測の処理時間に余裕を加えた値で設定し、コードにコメントで根拠を残す
- [ ] 各処理ステップの所要時間をログに記録し、タイムアウト設定の見直しに活用する
- [ ] Terraformのタイムアウト設定をデフォルト値のまま放置しない
