# 2026-04-10 OIDC認証設定ミス（2件）

## 概要

**SEV**: SEV-3

Cloud TasksとCloud Schedulerの2箇所でOIDC認証の設定ミスが発生。Cloud Tasksはサービスアカウントの指定誤り、Cloud Schedulerは不要なOIDC認証を付与したことで、それぞれジョブのキューイングと定期実行が失敗していた。

## タイムライン

**Cloud Tasks（4/10）**
- ジョブのキューイング後、Workerへのリクエストが認証エラーで失敗することを確認
- OIDCトークン生成に使用するサービスアカウントが誤っていたことが判明
- 正しいサービスアカウントに修正して解消

**Cloud Scheduler（4/10）**
- バッチ処理の定期実行を追加した直後から401エラーが継続発生
- Cloud Schedulerから`/batch-check`エンドポイントへのリクエストにOIDC認証を付けていたが、エンドポイント側がOIDC検証に対応していなかった
- `/batch-check`からOIDC認証ミドルウェアを削除して解消

## 根本原因

- **Cloud Tasks**: Terraform設定でサービスアカウントをコピーペーストした際に誤ったアカウントを指定
- **Cloud Scheduler**: OIDCを「つけるべき」と判断して実装したが、受け側のエンドポイントが対応していなかった。設計の確認なしに実装した

## 影響範囲

- **重大度**: SEV-3
- **影響ユーザー数**: 0人（開発環境のみ）
- **ダウンタイム**: Cloud Tasks: 数時間、Cloud Scheduler: 即日修正
- **TTD（検知まで）**: ともに即時（エラーログで確認）
- **TTR（復旧まで）**: ともに1時間以内
- **データ損失**: なし
- **外部影響**: なし

## 解決策

- Cloud Tasks: 正しいサービスアカウントに修正
- Cloud Scheduler: エンドポイントからOIDC認証ミドルウェアを削除

## 再発防止策

- [ ] 認証設定変更後は即座にエンドツーエンドで動作確認する
- [ ] サービスアカウントの指定はTerraform変数で一元管理し、コピーペーストミスを防ぐ
- [ ] 新しいエンドポイントに認証を追加する場合は、呼び出し元と受け側の両方の設定を事前に設計する
