# Postmortem

インシデントのポストモーテム集。プロダクト名・機密情報は記載しない。

## ファイル命名規則

`YYYY-MM-DD-<概要>.md` でルートにフラットに追加していく。

## 重大度（SEV レベル）

| レベル | 基準 |
|--------|------|
| SEV-1 | 本番の主要機能が全停止、データ損失あり |
| SEV-2 | 本番の主要機能が部分停止 |
| SEV-3 | 軽微な機能影響、回避策あり |
| SEV-4 | 開発・ステージングのみ、本番影響なし |

## テンプレート

```
## 概要

**SEV**: SEV-x

## タイムライン

## 根本原因

## 影響範囲

- **重大度**: SEV-x
- **影響ユーザー数**: x人 / x%
- **ダウンタイム**: xh xm
- **TTD（検知まで）**: xm
- **TTR（復旧まで）**: xm
- **データ損失**: あり / なし
- **外部影響**: あり / なし

## 解決策

## 再発防止策
```
